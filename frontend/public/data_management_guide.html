📊 Data Management Guide

Complete control over your datasets in the Semantic Sales Trend Analyzer

🔄 Data Management Options

When data is loaded, you now have three powerful options in the header:

1. 🔄 Refresh Data Status

What it does: Checks if data is still available in the backend

When to use: After backend restart or if you're unsure about data status

Result: Updates the "Data Loaded" indicator

2. 📤 Upload New Data

What it does: Replaces current dataset with a new CSV file

When to use: When you want to analyze different data

Result: Old data is replaced, all visualizations update automatically

3. 🗑️ Remove Dataset

What it does: Completely clears all uploaded data

When to use: When you want to start fresh or test with different data

Result: All data cleared, app returns to "Upload CSV" state

⚠️ Important Notes

  • Remove Action: Shows a confirmation dialog before deleting data
  • Data Persistence: Data persists across page refreshes but not backend restarts
  • File Format: Only CSV files are supported
  • Column Mapping: The app automatically maps common column names

✅ Quick Workflow Examples

Scenario 1: Test Different Datasets

  1. Upload initial dataset
  2. Analyze the results
  3. Click "New Data" to upload a different dataset
  4. Compare the new results

Scenario 2: Start Fresh

  1. Click "Remove" to clear current data
  2. Confirm the removal in the dialog
  3. Upload a fresh dataset
  4. Begin new analysis

Scenario 3: Backend Restart Recovery

  1. Notice data seems missing after backend restart
  2. Click "Refresh" to check data status
  3. If data is missing, upload again
  4. Continue with analysis

🎯 Best Practices

  • Backup Important Data: Keep copies of important CSV files
  • Test with Sample Data: Use the provided sample data to test features
  • Check Data Format: Ensure CSV has required columns (ProductName, Quantity, UnitPrice, etc.)
  • Use Refresh First: Try "Refresh" before re-uploading the same data
